# CL 26 — test: pick dual-transport-safe ports; surface Start errors (b/029)

**Bug:** b/029

## Description

While verifying CL 25 (an HTML-only dashboard change), the full test
run failed in `internal/dnsserver`: `TestServer_StartShutdownLifecycle`
and `TestIntegration_FullPipeline` timed out waiting for the freshly
started `dns.Server` to answer a probe — consistently, on a machine
where the same suite had passed hours earlier.

Root cause (full narrative in b/029): `pickFreePort` bound a free TCP
port and assumed the port number was also UDP-bindable. Windows
reserves large contiguous port ranges **per protocol** (Hyper-V/WSL
dynamic exclusions), and the TCP ephemeral allocator had drifted into
a UDP-excluded block after a VirtualBox VM started that afternoon —
so every picked port failed its UDP bind, and the test's
probe-timeout path swallowed the bind error because it never read the
`startErr` channel. The first fix attempt (allocate via UDP `:0`,
verify TCP) uncovered the mirror problem — the UDP allocator sat
inside a TCP-excluded block, and sequential `:0` allocation cannot
escape an 800-port exclusion within any reasonable retry budget.

Fix:

- `pickFreePort` probes **random** ports in 20000–47999 (below the
  49152+ dynamic-reservation area) and returns only a port that bound
  successfully for **both** UDP and TCP. Random probing escapes
  contiguous excluded blocks in one or two attempts; 20 attempts is
  the give-up budget.
- The lifecycle test's probe-timeout branch now drains `startErr` and
  includes what `Start` returned in the failure message, so a bind
  error can never masquerade as a timeout again.

Test-only change; no production code touched. Also a portability fix
in disguise: any CI runner or WSL host with Hyper-V port exclusions
could have hit this on an unlucky allocator position.

Reproduction of the original failure is environmental: it requires the
OS ephemeral allocator of one protocol to sit inside an
excluded-port-range of the other (`netsh int ipv4 show
excludedportrange` shows the live blocks).
