# CL 22 — s-hole: act on sixth staff review (T1–T8)

**Bug:** —

## Description

Acts on every finding from the sixth independent staff review. This
round found two genuine correctness bugs (T1, T2) that five earlier
rounds missed, plus six robustness/readability items.

**Correctness fixes (T1–T2)**

- **T1 (P1, config)** — `cache_size: 0` could not disable the cache
  from YAML, despite README, config.yaml, and the struct comment all
  promising it. `applyDefaults()` ran *after* the decode and treated
  `CacheSize == 0` as "unset", forcing it back to 2000 — a post-decode
  fixup cannot tell an explicit 0 in the file apart from an absent key.
  (`S_HOLE_CACHE_SIZE=0` worked only because env overrides apply after
  defaults.) Fix: seed `defaultCacheSize` / `defaultBlockTTL` onto the
  struct *before* `yaml.Decode`, and drop the two branches from
  `applyDefaults`. `block_ttl: 0` (legal DNS — "do not cache this
  reply") is honored by the same change. New regressions:
  `TestLoad_CacheSizeZeroDisables`, `TestLoad_BlockTTLZeroHonored`.
- **T2 (P1, dnsserver)** — truncated upstream replies were never
  retried over TCP. `forward()` used a UDP-only `dns.Client`, so a
  TC-flagged answer was relayed verbatim — and because inbound TCP
  queries were also forwarded over UDP, the client's own TCP fallback
  (promised by DESIGN.md and the package doc) looped straight back
  into the same truncated reply. Large answers (DNSSEC, big TXT/CDN
  RRsets) could never resolve. Fix: new `exchange()` helper does the
  UDP exchange and, on `resp.Truncated`, retries the same upstream over
  TCP; if the TCP retry fails the truncated UDP reply is returned (the
  upstream is demonstrably alive; TC + partial answer beats SERVFAIL).
  Both forwardWith sweeps now share it. `cache.Set` additionally
  refuses `msg.Truncated` so a partial answer is never replayed for its
  TTL. New tests: `TestForward_TruncatedUDPRetriesOverTCP`,
  `TestForward_TruncatedKeptWhenTCPUnavailable`,
  `TestCache_RejectsTruncated`, driven by a `startTruncatingUpstream`
  mock that truncates on UDP and answers fully on TCP (same port).

**Robustness fixes (T3–T6)**

- **T3 (P2, api)** — `/api/queries?limit=` was unbounded; one request
  could marshal the entire SQLite history into a single JSON response
  on the unauthenticated admin port. Extracted `parseLimit()` with a
  default of 50 and a clamp at 1000 (same defense-in-depth reasoning
  as the 64 KiB body cap). Table-driven `TestParseLimit`.
- **T4 (P3, main)** — the startup banner printed
  `Admin UI → http://<lan-ip>:8080` for every LAN interface even when
  `api_listen` was bound to `127.0.0.1` (the default), advertising a
  URL that is connection-refused for every other device.
  `printNetworkHint` now takes the API host and, when it is loopback
  (`isLoopbackHost`), prints a single
  `http://127.0.0.1:<port> (this machine only)` line. New tests:
  `TestPrintNetworkHint_LoopbackAPIPointsAtLocalhost`,
  `TestIsLoopbackHost`.
- **T5 (P3, blocklist)** — one line past `bufio.Scanner`'s default
  64 KiB token cap (a mis-served binary, a minified HTML error page)
  aborted the *entire* list with `ErrTooLong` even when every other
  line was fine. `parseHostsFormat` now raises the cap to 1 MiB;
  garbage lines are still dropped one at a time by `ValidDomain`.
  New `TestParseHostsFormat_LongLineDoesNotAbortList`.
- **T6 (P3, cache)** — the cache key used bare `dns.TypeToString` /
  `dns.ClassToString` map lookups, which render every *unknown* code
  as `""` — two distinct unknown qtypes collided on one key and could
  serve each other's cached answers. Switched to
  `dns.Type(...).String()` / `dns.Class(...).String()`, whose
  `TYPE1234` / `CLASS1234` fallbacks keep unknown codes distinct.
  New `TestCache_KeyDistinguishesUnknownTypes`.

**Readability (T7–T8)**

- **T7** — `TestApplyEnvOverrides_EnablePprof` carried a dead `want`
  table column, an `_ = tc.want` anchor, and a comment contradicting
  the row above it; the table now holds the real expectations
  (`{"", false}` included). `store_test.go` hand-rolled a 14-line
  `itoa` helper with a bogus "avoids strconv in the hot loop"
  justification — the strings are built before `b.ResetTimer()`;
  replaced with `strconv.Itoa` and deleted the helper.
- **T8** — `metrics.go` blanked an unused `hits` return with a
  trailing `_ = hits` line; now destructured as `_, misses, size :=`
  with the explanation moved above the call.

**Docs**

- README: coverage table refreshed (module-wide 71.6 % → 72.1 %),
  `/api/queries` row documents the 1000 cap, `block_ttl` row documents
  the 0 behaviour.
- DESIGN.md: upstream TCP-retry paragraph in the DNS server section;
  "never caches truncated" in the cache section; pre-decode default
  seeding note in the Configuration section; coverage paragraph
  refreshed; T1–T6 added to the regression-ID list.
- config.yaml: `block_ttl` comment documents 0.
- CONTRIBUTING.md: ID conventions now name the `R`/`S`/`T` rounds;
  coverage numbers refreshed.
- CHANGELOG: Fixed/Changed entries for T1–T6.
