# CL 27 — RFC 6303 local PTR answering for private-range zones

## Description

Implements roadmap item #9. PTR queries whose names fall under the RFC 6303
locally-served reverse zones are answered with authoritative NXDOMAIN by the
DNS handler instead of being forwarded upstream. A `local_ptr` config field
(default `true`) lets operators opt out if they run a private reverse DNS zone
on their LAN.

## Motivation

Observed during the 2026-07-12 VM deployment test: every `nslookup` probe
generated three query-log entries — a PTR reverse-lookup for the server's own
private IP, then the actual question. On a real home network this noise is
constant. Two compounding problems arise:

1. **Wasted round-trips**: the upstream always returns NXDOMAIN for private-range
   PTR queries. The response cache does not store NXDOMAIN, so every repeat query
   pays the full upstream round-trip again — there is no escape valve.
2. **Privacy leak**: forwarding `192.168.x.x` PTR queries tells the upstream
   resolver (Cloudflare, Google, ISP) which private addresses your LAN uses. The
   upstream cannot help, and the information has no legitimate recipient.

RFC 6303 (§4) says resolvers SHOULD answer these zones locally. Unbound,
dnsmasq, and systemd-resolved all do so by default.

## Design decisions

- **NXDOMAIN (not NODATA)**: RFC 6303 recommends NXDOMAIN for the whole zone;
  private reverse zones as a whole do not exist in the public DNS.
- **Separate `local_ptr_count` stat**: local PTR answers are not blocked (no
  blocklist involvement), not forwarded, and not cached. They count in
  `total_queries` (they are queries) but form their own outcome bucket, excluded
  from the cache-hit-rate denominator so the percentage stays meaningful.
- **`local_ptr: true` default**: safe for any LAN without a private reverse zone,
  which is the overwhelming majority of home deployments. Operators who run a
  local authoritative reverse zone can set `local_ptr: false` (or
  `S_HOLE_LOCAL_PTR=0`) to restore forwarding.
- **Check order**: local PTR interception is tested before the blocklist check,
  so private PTR queries are never counted as blocked even if someone adds an
  `in-addr.arpa` pattern to their blocklist.

## Files changed

- `internal/config/config.go` — `LocalPTR bool` field seeded to `true` before
  YAML decode (same pattern as `CacheSize`/`BlockTTL`); `S_HOLE_LOCAL_PTR` env
  override in `applyEnvOverrides`.
- `internal/stats/stats.go` — `localPTR atomic.Int64`, `RecordLocalPTR()`,
  `LocalPTRCount int64` in `Summary`; updated `Snapshot` excludes localPTR from
  the cache-hit denominator.
- `internal/dnsserver/handler.go` — `privateReverseZones` slice, `isPrivatePTR`
  helper, `localPTR bool` field on `Handler`, updated `NewHandler` signature,

- `cmd/s-hole/main.go` — passes `cfg.LocalPTR` to `NewHandler`.
- `config.yaml` — documents the `local_ptr` field.
- `README.md` — config table row and env-var table row; updated `/metrics` route
  description.
- `docs/ROADMAP.md` — item #9 marked `done (CL 27)`.

## Testing

- `TestIsPrivatePTR` — table-driven unit test covering all RFC 6303 ranges (10/8,
  16 172.x zones, 192.168/16, ULA, link-local), public-range non-matches, and
  non-PTR qtype fast-path.
- `TestServeDNS_LocalPTRReturnsNXDOMAIN` — NXDOMAIN returned, authoritative bit
  set, `LocalPTRCount=1`, `BlockedCount=0`, upstream never called.
- `TestServeDNS_LocalPTRDisabledForwardsUpstream` — with `localPTR=false` the
  same query reaches the upstream.
- `TestServeDNS_LocalPTRPreservesEDNS0` — OPT record echoed on local PTR replies.
- `TestServeDNS_NonPrivatePTRIsForwarded` — public-range PTR queries bypass the
  interception and reach the upstream even with `localPTR=true`.
- `TestCounter_RecordLocalPTR` — `LocalPTRCount` increments, `BlockedCount` stays 0.
- `TestCounter_LocalPTRExcludedFromCacheHitDenominator` — forwardable denominator
  correctly excludes localPTR entries.
- `TestLoad_LocalPTRDefaultTrue` — empty config → `LocalPTR=true`.
- `TestLoad_LocalPTRFalseHonored` — explicit `false` in YAML survives the decode.
- `TestApplyEnvOverrides_LocalPTR` — all seven env-var values (1/true/yes/0/false/no/"")
  behave consistently with `S_HOLE_ENABLE_PPROF`.
