# CL 21 — s-hole: act on fifth staff review (S1–S11)

**Bug:** —

## Description

Acts on every finding from the fifth independent staff review.

**Code fixes (S1–S4)**

- **S1** — `version.Short()` was exported but only called from its own
  test. Converted from a 3-string return (a Go anti-pattern that
  forced callers to memorise argument order) to a struct
  `version.Info{Version, Commit, BuildDate}`. `cmd/s-hole/main.go`
  now uses it for the startup-log fields, so the API has a real
  production caller and isn't dead code.
- **S2** — `internal/dnsserver/integration_test.go` had a
  `_ = atomic.LoadInt64 // keep atomic import alive even if test
  changes` line. The test already used `atomic.Int64.Load()` via
  `upstreamHits.Load()`, so the import was alive; the line was
  cargo-cult. Removed it and the `"sync/atomic"` import alongside.
- **S3** — same test had a hardcoded `time.Sleep(150 * time.Millisecond)`
  to wait for the SQLite flush tick. Replaced with a 2 s poll loop
  that exits as soon as the expected row count appears. Runs in
  ~30 ms locally; resilient on a slow CI runner.
- **S4** — `reloadFn` used two stacked defers
  (`defer reloadMu.Unlock(); defer reloadWG.Done()`) whose LIFO order
  released the WaitGroup *before* the mutex. Functionally harmless
  but counterintuitive. Collapsed into a single `defer func() {
  Unlock(); Done() }()` so the order matches the read order.

**Test gap fills (S5–S7)**

- **S5** — `TestDBLogger_DroppedOnChannelOverflow` previously only
  asserted that `Log()` returned promptly. It now also asserts
  `Dropped() >= 3000` after pushing 5 000 entries into a 1 000-slot
  channel that isn't draining (1 h flush interval). Companion test
  `TestDBLogger_DroppedZeroUnderNormalLoad` asserts the counter
  stays at zero under healthy load — a regression that miscounted
  by always incrementing would fail this one.
- **S6** — new `internal/api/pprof_test.go` with three tests:
  - `TestPprof_OffByDefault` — drives 7 pprof paths and asserts
    every one returns 404 (the SECURITY.md promise).
  - `TestPprof_OnWhenEnabled` — `EnablePprof(true)` + GET
    `/debug/pprof/` returns 200 and the index mentions `heap`.
  - `TestPprof_CmdlineWhenEnabled` — confirms named subroutes (not
    just the index) are reachable.
- **S7** — `TestRunTickerOnce_LogsPanicWithStack` redirects
  `slog.Default()` to an `io.Writer`, panics from a ticker fn, and
  asserts the captured output contains (a) the panic message,
  (b) a `stack=…` attribute, and (c) the runTickerOnce frame. R45
  is now regression-tested instead of just intended.

**Refactor (S8)**

- `runTicker` now takes `context.Context` as the first argument and
  exits when the ctx is cancelled. `cmd/s-hole/main.go` creates an
  application-wide `runCtx` and cancels it as the first step of
  `doStop`, so the stats-print and blocklist-refresh tickers
  unwind cleanly instead of being implicitly reclaimed by `os.Exit`.
  New test `TestRunTicker_StopsOnContextCancel` verifies the
  goroutine exits within 500 ms of cancel.

**Docs (S9–S11)**

- **S9** — `docs/CL.md` was a single 1 600-line file. Split into
  per-CL files under `docs/cls/CL-01.md` through `docs/cls/CL-20.md`
  via a one-shot `awk` script (no content loss; only the
  pre-existing 8-line preamble is dropped). The new `docs/CL.md` is
  a thin index table linking to each per-CL file. Future CLs add a
  new file plus an index row.
- **S10** — new `CONTRIBUTING.md` at the repo root. Covers the
  Makefile entry points, fuzz-run instructions, project structure,
  the `b/NNN` + `R NN` ID conventions, coverage targets, code-style
  expectations, and the doc-sync rule.
- **S11** — README gets a `### Contents` block right after the
  intro paragraph with anchored links to every major section plus
  a pointer to the maintainer-facing docs (DESIGN, CL, BUGS,
  CHANGELOG, CONTRIBUTING).
