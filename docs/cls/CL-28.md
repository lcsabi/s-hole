# CL 28 — Blocklist size in `/api/stats` and dashboard

## Description

Surfaces the current number of blocked domains in the `/api/stats` JSON
response and displays it as a fifth stat card on the dashboard.

## Motivation

The blocklist domain count was already exposed in two places — the startup
log line and `shole_blocklist_size` in `/metrics` — but neither the REST
API nor the web dashboard showed it. This is the most useful at-a-glance
signal that the blocklist downloaded, parsed, and survived the last refresh.
Without it, a failed refresh that leaves the block set empty is invisible
to anyone watching the dashboard.

Roadmap item #10; companion to the Cache Hit Rate card added in CL 25.

## Design decisions

**Where to plumb `store.Len()`**: the roadmap noted two options — add
`BlocklistSize` to `stats.Snapshot` or set it in the API handler. The
handler (`handleStats`) is the lighter touch: the `stats` package has no
reason to depend on the `blocklist` package, and `Server` already holds
`*blocklist.Store` for `/readyz`. `BlocklistSize` is declared on
`stats.Summary` (so JSON encoding and test assertions work naturally) but
is zero-valued from `Counter.Snapshot` and set by the caller.

**Dashboard layout**: the existing `.cards` grid used `repeat(4, 1fr)`.
Rather than hardcoding 5, the grid was updated to
`repeat(auto-fit, minmax(160px, 1fr))` — cards fill the row on wide
screens and wrap gracefully on narrow ones.
